# 全局指导

## 语言规范
- 所有文档、讨论与提交说明默认使用简体中文，除行业或框架的固定术语需保留英文外，不使用其他语言。

## 测试约定
- 若项目规范未强制指定其他语言，所有测试与测试用例一律使用 Go 编写。
- 优先使用 Go 1.25 版本执行测试。

## Context7 使用准则
- 需要官方库或框架资料时，优先使用 context7 MCP 服务，减少依赖不确定来源的信息。
- 调用 `context7__get-library-docs` 之前，必须先使用 `context7__resolve-library-id` 获取准确的 `/org/project[/version]` 标识；若用户已提供完整 ID，可直接使用。
- 遇到名称歧义或多个候选库时，在响应中说明筛选理由；若不确定用户需求，先向用户确认后再继续操作。
- 拉取文档时合理设置 `tokens` 与 `topic`，默认 5000 已满足日常需求，扩大范围需给出充分理由。
- 使用 context7 文档撰写答案时，确保内容与原文一致，并注明来源；若查无结果，应明确告知并说明尝试过的步骤。
- 在 context7 不能满足需求时，按流程记录问题，再考虑使用其他工具或请求用户提供更多信息。

## Chrome DevTools MCP 使用准则
- 需排查浏览器端行为、排版或网络问题时，优先调用 `chrome-devtools` MCP，而非手工描述页面状态。
- 首次使用前确认本机已安装并可访问 Chrome 浏览器；若连接失败，先检查浏览器版本与权限再行反馈。
- 通过 MCP 发起调试命令时，明确目标页面、期望采集的数据（如 DOM、Network、Console），避免重复抓取。
- 获取结果后及时整理关键观察、截取必要信息并引用 MCP 输出，避免遗漏上下文。
- 若 MCP 不支持所需操作或遇到异常，应记录已尝试的命令与错误信息，再咨询用户或改用其他工具。

## 代码读取与理解
- 尽可能使用并行方式读取与理解代码（例如并行检索关键字/符号、并行打开相关文件），以更快建立全局上下文并减少遗漏。

## 文档规范
- 除 `AGENTS.md` 外，所有文档必须归属于 `docs/` 目录。
- `docs/` 目录根下仅存放索引文件 `index.md`，所有具体文档放置于子目录中。
- 新增或变更文档时，需同步更新 `docs/index.md` 索引。
